## Objective

Create `openevolve/migrate_legacy.py` — a standalone idempotent script that moves all non-`runs/` subdirectories under `openevolve_output/` to `openevolve_output/runs/legacy/cavitydetection/` using `os.rename()`. Implement migration tests in `test_run_structure.py`.

Purpose: Preserves all existing experiment data while transitioning to the new run directory layout. Must handle all five current top-level dirs (`baseline/`, `best/`, `checkpoints/`, `logs/`, `prompt1/`) and any future additions.

Output: `migrate_legacy.py` + two green migration tests.

## Context

- D-03: Uses `os.rename()` — atomic on same filesystem (both src and dst are under `openevolve_output/` on `/dev/nvme0n1p2`). Per RESEARCH.md Pitfall 1: add a `shutil.move()` fallback for cross-device `OSError: [Errno 18]` in case `--output-root` points to another mount.
- D-04: Idempotent — if `dst` already exists, skip with a print message (per CONTEXT.md).
- RESEARCH.md verified: five directories present (`baseline/`, `best/`, `checkpoints/`, `logs/`, `prompt1/`). Migration scans `os.listdir()` excluding `{"runs"}` — no hard-coded list of two names.
- Migration target path: `openevolve_output/runs/legacy/cavitydetection/<dirname>/`
- `migrate_legacy.py` structural analog: `test_consolidation.py` shebang/docstring/SCRIPT_DIR pattern.
- File is callable standalone: `../openevolve/.venv/bin/python openevolve/migrate_legacy.py` from project root.

<action>
Create `openevolve/migrate_legacy.py` as a new file. Structure:

1. Shebang `#!/usr/bin/env python3`. Module docstring:
   ```
   Migrate legacy flat experiment directories to the new runs/ structure.

   Usage (from project root):
       ../openevolve/.venv/bin/python openevolve/migrate_legacy.py [--output-root PATH]

   Idempotent: already-migrated directories are skipped without error.
   Moves all non-runs/ subdirectories under OUTPUT_ROOT to:
       OUTPUT_ROOT/runs/legacy/cavitydetection/<dirname>/
   ```
2. Imports: `argparse`, `os`, `shutil`, `sys`. Module-level `SCRIPT_DIR = os.path.dirname(os.path.abspath(__file__))`.

3. Function `migrate_legacy(output_root: str) -> None`:
   - Docstring: `"""Move all non-runs/ subdirectories to runs/legacy/cavitydetection/."""`
   - `legacy_dir = os.path.join(output_root, "runs", "legacy", "cavitydetection")`
   - `os.makedirs(legacy_dir, exist_ok=True)`
   - `skip = {"runs"}` — only the `runs/` dir itself is excluded; all other subdirs move
   - Loop `for name in sorted(os.listdir(output_root))`: skip if `name in skip`; skip if not `os.path.isdir(src)`; compute `dst = os.path.join(legacy_dir, name)`; if `os.path.exists(dst)`: print `f"Skip (already migrated): {name}"` and continue
   - Move: first try `os.rename(src, dst)`. On `OSError` with errno 18 (EXDEV / cross-device link), fall back to `shutil.move(src, str(dst))`. Print `f"Moved: {name} -> runs/legacy/cavitydetection/{name}"` on success.

4. Function `main()` with argparse:
   - One arg: `--output-root`, default `os.path.join(SCRIPT_DIR, "openevolve_output")`, help text describes the default.
   - Validate `output_root` exists; if not: `sys.exit(f"Output root not found: {output_root}")`.
   - Call `migrate_legacy(args.output_root)`.
   - Print `"Migration complete."` at the end.

5. `if __name__ == "__main__": main()`
</action>

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| CLI → migrate_legacy.py | `--output-root` is researcher-supplied and becomes a filesystem root for rename operations |
| migrate_legacy() → os.rename | Moves entire directories; if path is wrong, data could land in unintended location |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-05 | Tampering | `--output-root` → `legacy_dir` construction | accept | Script is run manually by the researcher against their own data directory. `output_root` existence is validated before proceeding (`sys.exit` if absent). Risk of misdirection is very low for a local research tool. |
| T-03-06 | Tampering | `os.rename` moves data to `legacy_dir` | mitigate | The `skip = {"runs"}` guard prevents the `runs/` directory from being moved into itself. The existence check on `dst` prevents overwrites of already-migrated data. |
| T-03-07 | Denial of Service | Migration fails mid-run, leaving partial state | accept | `os.rename` is atomic per directory. If it fails, source is intact. Each dir is moved independently — a failure on one does not affect others. Researcher can re-run safely due to idempotency (D-04). |
| T-03-08 | Tampering | Cross-device `os.rename` falls back to `shutil.move` (copy+delete) | accept | Fallback only triggered on EXDEV errno. `shutil.move` preserves file contents. If interrupted mid-copy, source files remain (no data loss, but partial destination). Acceptable for researcher workflow. |
</threat_model>
