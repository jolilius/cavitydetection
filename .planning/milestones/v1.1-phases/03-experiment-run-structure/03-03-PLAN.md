## Objective

Update `show_results.py` and `show_consolidated.py` to work with the new `runs/<run_id>/cavitydetection/<prompt>/` directory layout, add `--run` filter arguments to both, wire `RUN=` pass-through into the Makefile display targets, and update `evolve-all` to generate a single `RUN_ID` before the prompt loop.

Purpose: Delivers the researcher-facing display and run-grouping capability. After this plan, `make evolve-all` produces a single named run and `make show-results RUN=<id>` filters to it.

Output: Two updated display scripts, an updated Makefile, and four green tests covering display filtering, consolidated filtering, and run grouping.

## Context

- D-05: No-arg `make show-results` → all runs merged, flat by prompt (backward-compatible) (per CONTEXT.md)
- D-06: `make show-results RUN=<id>` → filtered to that run (per CONTEXT.md)
- RESEARCH.md Pitfall 2: `os.listdir(OUTPUT_ROOT)` in `show_results.py` line 95 breaks after migration because `OUTPUT_ROOT` contains only `runs/`. Must replace with `glob.glob` walk.
- RESEARCH.md Pattern 5: replacement discovery using `glob.glob(os.path.join(output_root, "runs", "**", "results.json"), recursive=True)`; parse run_id and prompt from path parts.
- `show_consolidated.py` already uses `load_all_results()` which uses `os.walk()` — only the root path argument needs updating to `openevolve_output/runs/`. A `run_id` column must be added to filter by.
- RESEARCH.md Pattern 6 Option B: Makefile uses Python subprocess to call `generate_run_id` from `run_experiment.py` — centralizes sanitization logic, avoids shell sed quoting.
- RESEARCH.md Pattern 7: `$(if $(RUN),--run $(RUN),)` in Makefile recipes.
- `test_evolve_all_shared_run` is an integration test — it does NOT spawn actual OpenEvolve. It creates a fake tempdir structure and verifies the path logic, not the actual subprocess.

**A. Replace `evolve-all` recipe** — replace lines 163-172 with a version that generates `RUN_ID` once using the Python helper from `run_experiment.py`, then passes `--run $$RUN_ID` to each per-prompt invocation:

```makefile
evolve-all:
	@echo "Running all prompt variants (explanations enabled by default)..."
	@echo "To skip explanation generation, use: EXPLAIN_GENERATIONS=0 make evolve-all"
	@RUN_ID=$$($(OPENEVOLVE_PYTHON) -c "import sys; sys.path.insert(0,'openevolve'); from run_experiment import generate_run_id; print(generate_run_id('openevolve/config.yaml'))"); \
	echo "Run ID: $$RUN_ID"; \
	echo "Running experiments for prompts: $(PROMPT_NAMES)"; \
	for p in $(PROMPT_NAMES); do \
		echo ""; \
		echo "=== Experiment: $$p ==="; \
		$(OPENEVOLVE_PYTHON) openevolve/run_experiment.py $$p \
			--run $$RUN_ID \
			--iterations $${ITERATIONS:-80}; \
	done
```

Key points:
- `RUN_ID` is computed once outside the loop using `$(OPENEVOLVE_PYTHON) -c "..."` subprocess to `generate_run_id`
- The `--run $$RUN_ID` flag is passed to each `run_experiment.py` invocation
- `$$RUN_ID` uses double-dollar shell escaping (existing convention in this Makefile)
- The `$${ITERATIONS:-80}` default pattern is kept (existing convention)
