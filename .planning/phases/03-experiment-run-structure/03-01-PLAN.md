## Objective

Create `openevolve/test_run_structure.py` with stub tests (Wave 0), then extend `openevolve/run_experiment.py` with the `generate_run_id()` function, `write_run_metadata()` function, `--run` and `--output-root` CLI args, and the revised output path construction.

Purpose: Establishes the run ID contract and directory layout that all subsequent plans depend on. Tests fail (red) after Wave 0 and turn green after this plan's implementation tasks complete.

Output: A working `run_experiment.py` that writes to `runs/<run_id>/cavitydetection/<prompt>/` and records `metadata.json`.

- D-01: Run ID format `YYYY-MM-DD_HHMM_<sanitized-model>` (per CONTEXT.md)
- D-02: Sanitize model with `re.sub(r'[^a-zA-Z0-9-]+', '-', model)` then `re.sub(r'-+', '-', s).strip('-')` (per CONTEXT.md)
- D-07: No `--run` → auto-generate; no fallback to legacy flat path (per CONTEXT.md)
- D-08: `--output-root` overrides the default `openevolve/openevolve_output/` (per CONTEXT.md)
- The existing `config` dict from `yaml.safe_load` is already in scope — reuse it for both the LLM model name and `config_snapshot` in metadata.json
- `consolidate_experiment(output_dir=output_dir, ...)` call stays; `output_dir` now points to the prompt-level dir inside the run tree
- Existing non-blocking `try/except + print("Warning: ...")` pattern applies to `write_run_metadata()` call
- `test_consolidation.py` is the exact structural analog for the test file

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| CLI → run_experiment.py | `--run` and `--output-root` are researcher-supplied strings that become filesystem paths |
| run_experiment.py → filesystem | `output_dir` and `run_dir` are constructed from user input; must not escape the intended output root |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-01 | Tampering | `--run` arg → `run_dir` path | mitigate | `generate_run_id()` sanitizes auto-generated IDs to `[a-zA-Z0-9-_]+`. For user-supplied `--run` values, the path is constructed with `os.path.join(output_root, "runs", run_id)` — a `run_id` of `../../etc` would produce `output_root/runs/../../etc` = `output_root/../etc`. Add validation: after constructing `run_dir`, assert `os.path.abspath(run_dir).startswith(os.path.abspath(output_root))`. If it does not, exit with an error message. |
| T-03-02 | Tampering | `--output-root` arg | accept | Researcher-only local CLI tool. No sensitive data outside the project directory. Risk of a researcher misdirecting their own output is very low. Document the flag as "for advanced use only". |
| T-03-03 | Information Disclosure | `config_snapshot` in metadata.json | accept | Config contains model names and iteration counts — no secrets (API keys live in env vars not config.yaml). Acceptable for a local research tool. |
| T-03-04 | Denial of Service | Repeated write_run_metadata calls racing | accept | Single-researcher workflow; no concurrent writes expected. The read-then-merge pattern is sufficient. |
</threat_model>
