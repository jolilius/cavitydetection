<task type="auto">
  <name>Task 1: Design and document explanation prompt</name>
  <files>openevolve/explanation_prompt.txt, openevolve/EXPLANATION_DESIGN.md</files>
  <action>
Create two artifacts:

1. **openevolve/explanation_prompt.txt** — A system message that will be sent to the LLM to explain code transformations. The prompt should:
   - Describe the task: "You will receive two C programs (evolved and baseline), explain the key optimization strategy in 1-2 sentences"
   - Provide context: Show an example of what baseline code looks like (from initial_program.c) and what an evolved version might look like
   - Specify output format: Exactly 1-2 sentences, focused on memory/cache optimization techniques (loop reordering, loop fusion, buffering, etc)
   - Avoid generic praise or speculation; focus on *what* was changed, not *why* the LLM did it
   - Include hard constraints: no bullet lists, no markdown, plain text only

2. **openevolve/EXPLANATION_DESIGN.md** — A design document explaining:
   - Prompt structure and rationale
   - What makes a "good" explanation (concise, specific, optimization-focused)
   - Example explanations (manually written) showing desired output quality
   - Known limitations (LLM may hallucinate; explanations are subjective; not guaranteed correct)
   - **Prompt Versioning Strategy:** How to iterate the prompt safely without losing reproducibility
     - Version numbering scheme (semantic versioning: major.minor)
     - How prompt_version and prompt_hash track changes
     - When to regenerate explanations (if we change the prompt, re-run experiments to re-capture with new version)
     - How to compare explanations across prompt versions using the results data
     - Git commit message guidance for prompt changes

The prompt should be crafted to elicit 1-2 sentence explanations that a researcher can quickly scan to understand the transformation strategy. Avoid asking for code diffs (too long); focus on natural language explanation.

**3. Add version header to explanation_prompt.txt:**
At the top of the prompt file, include a version marker:
```
# Explanation Prompt v1.0
# Last modified: 2026-05-13
# SHA256: [will be computed at runtime]

[Rest of prompt content...]
```

This marker:
- Documents prompt version for reference
- Helps with git history (easy to find when it was changed)
- Is NOT parsed by code; purely for human reference
- The actual version is tracked in results via prompt_version and prompt_hash

<task type="auto">
  <name>Task 2: Implement explanation generator module</name>
  <files>openevolve/explanation_generator.py</files>
  <action>
Create a reusable Python module to generate explanations via LLM. The module should provide:

**Public API:**
```
generate_explanation(
    evolved_code: str,          # Full evolved C program
    baseline_code: str,         # Full baseline C program (from initial_program.c)
    llm_config: dict,           # Dict with keys: primary_model, api_base, api_key, temperature, max_tokens, timeout
    explanation_prompt_text: str # Loaded from explanation_prompt.txt
) -> str | None
```

Returns: A 1-2 sentence explanation string, or None if generation fails (timeout, API error, etc).

**Implementation details:**
- Use requests library to call OpenAI-compatible API (Ollama endpoint from config)
- POST to `{api_base}/chat/completions` with standard format
- Include full programs as context (not diffs)
- Request response format: `{ "choices": [{ "message": { "content": "..." } }] }`
- Timeout handling: catch exceptions, return None on timeout/500 errors (non-blocking)
- Logging: print to stdout for user visibility (e.g., "Generating explanation... OK" or "explanation timeout, skipping")
- No retry logic needed (failures are acceptable for now)

**Testing harness in main:**
```python
if __name__ == "__main__":
    import json
    import yaml
    
    # Load config and prompt
    with open("config.yaml") as f:
        config = yaml.safe_load(f)
    with open("explanation_prompt.txt") as f:
        prompt_text = f.read()
    
    # Dummy evolved/baseline for testing
    baseline = open("initial_program.c").read()
    evolved = baseline.replace("ComputeEdges", "ComputeEdgesOptimized")  # minimal diff
    
    # Call generator
    exp = generate_explanation(evolved, baseline, config["llm"], prompt_text)
    if exp:
        print(f"Generated: {exp}")
    else:
        print("Failed to generate explanation")
```

The module should be importable by both the integration task (02-02) and testable in isolation.
