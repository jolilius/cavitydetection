## Deliverables

### Task 1: Integrate Explanation Generation into run_experiment.py

**Modified:** `openevolve/run_experiment.py` (+120 lines)

**Changes:**
- Import explanation_generator module (graceful fallback if unavailable)
- Add `_generate_explanations_for_experiment()` helper function that:
  - Loads best_program_info.json from OpenEvolve output
  - Extracts evolved code source
  - Calls `generate_explanation()` with baseline code, evolved code, LLM config, and prompt
  - Returns dict mapping iteration number -> explanation text
- Load baseline code (initial_program.c) and explanation prompt at startup
- After OpenEvolve completes, call explanation generation (if enabled)
- Support EXPLAIN_GENERATIONS environment variable for opt-out:
  ```python
  if os.environ.get("EXPLAIN_GENERATIONS", "1") != "0":
      # Generate explanations
  ```
- Pass explanations dict to `consolidate_experiment()` with prompt_text and version
- Add timing output: "Generated N explanations in X.XXs"
- Non-blocking error handling: warnings logged, but experiments succeed even if explanations fail

**Key Features:**
- Graceful degradation: if explanation_generator module unavailable or LLM down, experiment continues
- Automatic: researchers don't need to do anything; explanations generated by default
- Configurable: EXPLAIN_GENERATIONS=0 disables explanation generation
- Integrated: explanations stored in results.json via consolidate_experiment()

**API Integration:**
```python
generate_explanation(
    evolved_code=evolved_code,
    baseline_code=baseline_code,
    llm_config=config["llm"],
    explanation_prompt_text=explanation_prompt
) -> Optional[str]
```

### Task 2: Update results_loader.py to Expose Explanations

**Modified:** `openevolve/results_loader.py` (+72 lines)

**Changes:**
- Update `load_results()` docstring:
  - Document explanation column in Returns section
  - Add note about NaN values for Phase 1 results (backward compatibility)
  - Add example showing how to view explained iterations
- Include explanation column in DataFrame column reordering
- Update `load_all_results()` docstring with explanation examples
- Add `get_explanations(df)` utility function:
  - Groups explanations by iteration and prompt
  - Returns dict: {iteration: {prompt: explanation_text}}
  - Handles missing explanations gracefully (skips NaN rows)
  - Useful for comparing explanation quality across prompt variants

**Backward Compatibility:**
- Explanation column may be NaN for Phase 1 results (generated before Wave 2)
- `df[df['explanation'].notna()]` filters to explained rows only
- All existing analysis code works unchanged

**Example Usage:**
```python
from results_loader import load_all_results, get_explanations

df = load_all_results()
# View all explained iterations
explained = df[df['explanation'].notna()]
print(explained[['iteration', 'prompt', 'improvement_percent', 'explanation']])

# Analyze explanations by iteration
by_iter = get_explanations(df)
for iteration, prompts in sorted(by_iter.items()):
    for prompt, explanation in prompts.items():
        print(f"Iter {iteration} ({prompt}): {explanation[:60]}...")
```

## Integration Flow

### Experiment Execution Flow (with explanations)

```
1. run_experiment.py invoked with prompt name
   ↓
2. Load baseline code (initial_program.c)
3. Load explanation prompt (explanation_prompt.txt)
4. Load LLM config (config.yaml)
   ↓
5. Run OpenEvolve (subprocess, unchanged)
   ↓
6. Check EXPLAIN_GENERATIONS environment variable
   - If "0": skip to step 8
   - If "1" (default): continue
   ↓
7. Generate explanations:
   a. Load best_program_info.json
   b. Extract evolved code
   c. Call generate_explanation() with baseline and evolved code
   d. Store result in explanations dict
   ↓
8. Consolidate results:
   a. Call consolidate_experiment() with:
      - output_dir, program, prompt_variant, baseline_accesses
      - explanations dict (may be empty)
      - explanation_prompt_text
      - prompt_version ("1.0")
   b. Write results.json with explanation field and explanation_config
   ↓
9. Results stored in openevolve_output/{prompt}/results.json
```

### Results DataFrame Access

```
df = load_results('openevolve_output/baseline/results.json')
# or
df = load_all_results()

# View explanation column
print(df[['iteration', 'explanation']])

# Filter to explained iterations
explained = df[df['explanation'].notna()]

# Analyze by iteration
from results_loader import get_explanations
by_iter = get_explanations(df)
```

## Known Limitations

1. **Single Iteration Explanations**
   - Current implementation extracts explanation only for the best solution found
   - Future work: extract explanations for all iterations (if OpenEvolve exposes per-iteration best results)
   - Mitigation: sufficient for current research goals; explanations of best solution are most valuable

2. **LLM Hallucination Risk**
   - Inherited from Wave 1 design
   - Explanations are advisory, not authoritative
   - Researchers should verify against actual code

3. **Ollama Availability**
   - Explanation generation requires Ollama running (optional, non-blocking)
   - Connection timeouts and errors are caught and logged
   - Experiments continue even if LLM unavailable

4. **Explanation Latency**
   - Each explanation adds ~1-2 seconds
   - For short test runs (10 iterations), acceptable
   - For long production runs (80+ iterations), use EXPLAIN_GENERATIONS=0 if needed
